# Leafstep Website Acceptance Criteria

Use this checklist to review the Codex implementation.

## Scope

- [ ] The website is informational only.
- [ ] There is no login.
- [ ] There is no account creation.
- [ ] There is no web dashboard.
- [ ] There is no check-in flow on the web.
- [ ] There is no weight tracking on the web.
- [ ] There is no goals management on the web.
- [ ] There is no backend.
- [ ] There is no database.
- [ ] There is no auth.
- [ ] There is no AI coach.
- [ ] There is no calorie, meal, macro, barcode, or food database feature.

## Content

- [ ] Uses the public brand `Leafstep`.
- [ ] Clearly says Leafstep is an iOS app.
- [ ] Clearly communicates consistency-first positioning.
- [ ] Mentions no calorie tracking.
- [ ] Mentions local-only / private design.
- [ ] Explains Today check-ins.
- [ ] Explains Flex Days.
- [ ] Explains Progress trends.
- [ ] Explains Goals.
- [ ] Explains History.
- [ ] Explains gentle achievements.
- [ ] Includes FAQ.
- [ ] Includes final CTA.

## Tone

- [ ] Copy is calm.
- [ ] Copy is supportive.
- [ ] Copy is non-shaming.
- [ ] Copy avoids aggressive fitness language.
- [ ] Copy avoids medical claims.
- [ ] Copy avoids guaranteed results.

## Visual design

- [ ] Warm ivory background.
- [ ] Soft mint accent/glow.
- [ ] Neutral white cards.
- [ ] Green used intentionally, mostly for primary CTA.
- [ ] Slate/neutral text.
- [ ] Lavender used for Flex Day accents.
- [ ] Blue used for weigh-in/informational accents.
- [ ] Orange used for streak accents.
- [ ] Page feels premium and relaxing.

## UX

- [ ] Page is responsive.
- [ ] Mobile layout works well.
- [ ] CTA appears early on mobile.
- [ ] Header is simple.
- [ ] Hero explains the product quickly.
- [ ] Sections are scannable.
- [ ] Static app mockups are clearly previews, not interactive app modules.

## Accessibility

- [ ] Semantic HTML is used.
- [ ] Headings are hierarchical.
- [ ] Links and buttons have accessible names.
- [ ] Contrast is readable.
- [ ] Decorative images are hidden from screen readers when appropriate.
- [ ] Motion respects `prefers-reduced-motion`.

## Technical

- [ ] No unnecessary dependencies added.
- [ ] No backend packages added.
- [ ] No auth packages added.
- [ ] No data persistence added.
- [ ] Build/lint passes, if applicable.
- [ ] README or summary explains how to preview the site.
