# Leafstep Informational Website Brief

Last updated: 2026-06-14

## 1. Project goal

Create a simple, premium, calming informational website for the Leafstep iOS app.

The website exists to explain what the app is, why it exists, and how it helps users. It should not recreate the app in the browser.

## 2. Critical scope rule

This is a **marketing / informational website only**.

Do not build a web app.

Do not build any browser-based version of the Leafstep product.

Do not build any user-facing tracking functionality on the website.

The website may show screenshots, mockups, static UI previews, and app feature explanations, but users should not be able to:

- Create an account
- Log in
- Track their weight
- Complete check-ins
- Manage goals
- View real progress data
- Save anything
- Sync anything
- Use a dashboard

## 3. Product summary

Leafstep is a local-only native iOS habit tracker for personal weight-loss consistency.

The app is focused on consistency, not calorie tracking.

Core promise:

> Track the days that matter. Build consistency. Watch the weight trend follow.

## 4. Website purpose

The landing page should:

- Explain Leafstep in a calm, clear way.
- Position it as an iOS app.
- Show the app’s current core features.
- Emphasize privacy and local-only tracking.
- Make the app feel premium, gentle, and trustworthy.
- Drive one simple CTA: join beta or download on the App Store.

## 5. Public brand

Use:

> Leafstep

Note: the native app project may still use the internal app name `Weigh` in places, but the public website should use `Leafstep`.

## 6. Primary positioning

Leafstep is a calmer way to stay consistent with your weight-loss habits.

It helps users check in, build consistency, plan Flex Days, track weight trends, see progress, and stay aware without pressure.

## 7. Target audience

Leafstep is for people who:

- Want to lose weight but do not want to count calories.
- Already have a diet or plan and need consistency tracking.
- Feel overwhelmed by detailed fitness apps.
- Want accountability without shame.
- Want to see the relationship between habits and weight trend.
- Want planned flexibility instead of all-or-nothing streaks.

## 8. Emotional promise

Leafstep should feel like:

- A calm daily rhythm.
- A private progress journal.
- A small check-in at the end of the day.
- A supportive reminder to keep going.

It should not feel like:

- A diet challenge.
- A medical app.
- A fitness influencer product.
- A calorie-counting machine.
- A social accountability platform.
- A web dashboard.

## 9. Core product pillars

### 1. Daily check-ins

Users complete a simple daily check-in from the Today tab.

Normal days support:

- Yes
- Mostly
- No

Check-ins may also capture:

- Movement
- Optional weight

### 2. Flex Days

Flex Days are planned breaks.

They pause streaks instead of breaking them.

Use language like:

> Planned break.
> Keep it intentional.
> Streak paused, not broken.

### 3. Progress trends

The app shows:

- Current weight
- Total lost
- Goal weight
- Weight trend chart
- Body measurements trend
- Monthly consistency

### 4. Simple goals

The app has core goals for:

- Following diet X days per week
- Moving X days per week
- Logging weight X times per week
- Reaching target weight

### 5. History

The app shows a monthly calendar with:

- On-plan days
- Mostly days
- Missed days
- Flex Days
- Weigh-ins

### 6. Gentle achievements

The app includes calm, local-only achievements for consistency milestones.

Examples:

- First check-in
- First weight log
- Steady week
- Trend builder
- Movement rhythm
- Year of small steps

### 7. Privacy

The app is local-only.

Use privacy as a major trust point:

- No account
- No social feed
- No cloud dashboard
- No food database
- No calorie tracking
- No web app

## 10. Non-goals

The website must not claim or imply support for:

- Web check-ins
- Web dashboard
- Cloud sync
- User accounts
- Backend
- Social features
- AI coaching
- Calories
- Meals
- Macros
- Barcode scanning
- Food database
- Wearable sync
- Medical advice
- Guaranteed weight loss

## 11. Tone

The copy should be:

- Calm
- Supportive
- Clear
- Premium
- Non-shaming
- Human
- Minimal

Avoid aggressive phrases like:

- Crush your goals
- No excuses
- Burn fat fast
- Transform your body
- Never miss a day again
- Your personal AI coach
- Track every bite
- Hack your metabolism

## 12. Recommended CTA strategy

Before App Store launch:

Primary CTA:

> Join the beta

Secondary CTA:

> See how it works

After App Store launch:

Primary CTA:

> Download on the App Store

Secondary CTA:

> See how it works

If there is no beta signup ready yet, use a placeholder CTA or mailto link and clearly mark it in code as replaceable.
