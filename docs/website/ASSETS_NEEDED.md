# Leafstep Website Assets Needed

This page can be built with CSS mockups first, then improved with real assets later.

## Required assets

### 1. Logo / app icon

Use the existing Leafstep app icon or splash artwork if available.

Needed formats:

- SVG preferred for logo mark
- PNG acceptable
- Transparent background preferred

Suggested names:

```txt
/website/assets/leafstep-logo.svg
/website/assets/leafstep-icon.png
```

### 2. App screenshots

Ideal screenshots:

1. Today screen
2. Progress screen
3. History screen
4. Flex Day state
5. Achievements bottom sheet or card

Suggested names:

```txt
/website/assets/screens/today.png
/website/assets/screens/progress.png
/website/assets/screens/history.png
/website/assets/screens/flex-day.png
/website/assets/screens/achievements.png
```

If real screenshots are not ready, use CSS mockups.

### 3. App Store badge

Use only when the App Store link is ready.

Suggested name:

```txt
/website/assets/app-store-badge.svg
```

If the app is not on the App Store yet, use `Join the beta` instead.

## Optional assets

### Abstract leaf path

A subtle abstract illustration can help the page feel branded.

Suggested:

- SVG path
- Very low contrast
- Mint or warm beige

### Achievement badges

Use static decorative chips or small badge graphics.

Suggested badge labels:

- First check-in
- Trend builder
- Flex Day saved

## Screenshot guidance

Screenshots should avoid personal data.

Use demo data only.

Good demo values:

- Current: `214.2 lb`
- Goal: `195 lb`
- Total lost: `8.6 lb`
- Weekly goal: `3 / 5`
- Streak: `4 days`
- Reminder: `Off`
- Flex Days: `Sat, Sun`

## Asset fallback

If assets are unavailable:

- Build phone mockups using HTML/CSS.
- Use simple charts and calendar dots as static visuals.
- Keep everything clearly decorative and non-interactive.
