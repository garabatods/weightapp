# Leafstep Landing Page Structure

## Page type

Single-page informational website.

This is not a web app.

## Route

Use the root route:

```txt
/
```

If using a separate static folder:

```txt
/website/index.html
```

## Recommended section order

1. Header
2. Hero
3. Problem / contrast
4. How it works
5. Feature grid
6. Privacy / local-only
7. Gentle reminders
8. FAQ
9. Final CTA
10. Footer

## Layout notes

### Header

- Keep minimal.
- Sticky header is okay if subtle.
- Use translucent warm background if sticky.
- Mobile can hide nav links and keep logo + CTA.

### Hero

Desktop:

- Two columns.
- Copy left.
- Static phone mockups right.

Mobile:

- Copy first.
- Phone mockups below.
- CTA above the fold.

### Problem cards

Use 3 cards.

Each card should have:

- Small icon bubble
- Short title
- One-sentence body

### How it works

Use 3 steps.

Avoid making this feel like onboarding for the web. It is only explaining the iOS app.

### Feature grid

Use 6 cards.

Each card should feel like a product feature preview, not a functional widget.

### Privacy section

Make this section visually distinct.

Suggestion:

- Large rounded card
- Soft mint background
- Lock or leaf icon
- Direct copy about local-only design

### FAQ

Can be accordion or static list.

Static list is simpler and acceptable.

### Footer

Footer should reinforce:

> Leafstep is a native iOS app. This website is informational only.
